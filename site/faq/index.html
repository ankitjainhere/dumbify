FAQ

Common questions about Dumbify.

What data does Dumbify collect?

None. There is no backend, no analytics, no crash reporting, no accounts. Everything — block rules, PINs, audit log — stays on the device. The only network call the app makes is to WorkManager scheduling (local only). The installer site fetches the APK from GitHub Releases; that's a standard GitHub CDN request.

Another Device Owner is already set — what do I do?

Android allows exactly one Device Owner at a time. If another MDM or parental control app is already set as DO, you must factory-reset the phone before Dumbify can be installed.

  1. Back up anything important (factory reset wipes the phone)
  2. Power off → hold Volume Down + Power to enter bootloader (varies by device)
  3. Select Wipe data / Factory reset
  4. After reset, run the Dumbify installer before signing into any Google account

I forgot the PIN and need to remove Dumbify

There is no PIN recovery. This is intentional — a recovery mechanism would let you bypass the protection.

The only option is a factory reset via the bootloader:

  1. Power off the phone
  2. Hold the bootloader key combo for your device (usually Volume Down + Power)
  3. Select Wipe data / Factory reset and confirm

This erases all data. If you used Google Backup, most app data restores automatically after signing in again.

This is why the setup wizard strongly recommends having a trusted person set the Removal PIN — someone who can make the removal decision consciously, not impulsively.

Why must I remove Google accounts before installing?

Android's DevicePolicyManager.setDeviceOwner() API refuses to run if any accounts (Google, Samsung, etc.) are present. This is an Android security restriction, not something Dumbify can bypass. You can add accounts back immediately after the install wizard completes.

What happens after too many wrong bypass PIN attempts?

3 wrong entries trigger a 5-minute cooldown. The counter and cooldown timestamp are stored in EncryptedSharedPreferences and survive reboots. After the cooldown you get 3 more attempts.

Does Dumbify survive a reboot?

Yes. A BootReceiver restarts AppMonitorService on every boot. Grant expiry (temporary unblocks) are re-enforced by ReSuspendWorker via WorkManager.

Can I open Settings / Phone / Camera even if they're blocked?

Yes. Phone, Messages, Camera, Settings, and Dumbify itself are hardcoded as always-allowed regardless of mode. You cannot lock yourself out of essentials.

What license is Dumbify under?

GPLv3. You can read, modify, and redistribute the source. Modified versions must also be GPLv3. See LICENSE.

Is there an iOS version?

No. iOS doesn't expose an equivalent of Device Owner mode to third-party apps. A meaningful iOS port isn't feasible. See future work.

Does the background monitoring drain battery?

AppMonitorService polls UsageStatsManager every ~500ms. In practice this is negligible — UsageStats queries are cheap cached reads. Real-world impact is expected to be <1% additional drain. Battery impact will be formally measured in v1 testing.
