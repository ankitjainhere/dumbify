About

Open-source, no backend, built to last.

What is Dumbify?

Dumbify is an Android app that uses the Device Owner API to hard-block distracting apps at the OS level. Unlike screen-time apps that rely on self-reported timers or accessibility hacks, Dumbify suspends apps the same way enterprise MDM software does — the app simply cannot be opened.

The goal is to make your phone boring on purpose. Not a timer. Not a nudge. A hard block with deliberate friction to remove.

Why Device Owner?

Most distraction-blocker apps can be bypassed by uninstalling them or disabling their accessibility service in Settings. Device Owner mode prevents uninstall (setUninstallBlocked) and gives access to setPackagesSuspended — the only API that prevents an app from launching at all, even from other apps.

The tradeoff: setup requires a desktop browser and a USB cable. That friction is intentional — it makes the commitment feel real.

Privacy

No data leaves your device. No accounts. No analytics. No crash reporting. The only outbound network calls are:

  • The installer site fetching the APK from GitHub Releases (a standard CDN request to GitHub)
  • WorkManager scheduling — fully local, no network

Tech stack

  • Kotlin + Jetpack Compose
  • Hilt (dependency injection)
  • Room (local database)
  • WorkManager (grant expiry scheduling)
  • EncryptedSharedPreferences + Argon2id (PIN storage)
  • DevicePolicyManager / Device Owner API

Installer site

  • Plain HTML + vanilla JS — no framework
  • ya-webadb for WebUSB ADB
  • Hosted on GitHub Pages
