Documentation

Setup, usage, and device-specific guidance.

Requirements

  • Android 10 or newer (SDK 29+)
  • No Google or other accounts signed in at install time
  • USB cable (data-capable, not charge-only)
  • Desktop Chrome or Edge for the WebUSB installer
  • USB debugging enabled (see below)

Enabling USB debugging

Stock Android (Pixel, Android One)

  1. Settings → About phone
  2. Tap Build number 7 times
  3. Settings → System → Developer options
  4. Enable USB debugging

Samsung (One UI)

  1. Settings → About phone → Software information
  2. Tap Build number 7 times
  3. Settings → Developer options
  4. Enable USB debugging

Xiaomi (MIUI / HyperOS)

  1. Settings → About phone
  2. Tap MIUI version 7 times
  3. Settings → Additional settings → Developer options
  4. Enable USB debugging

OnePlus (OxygenOS)

  1. Settings → About device
  2. Tap Build number 7 times
  3. Settings → System → Developer options
  4. Enable USB debugging

Removing accounts before install

Android's Device Owner API requires no accounts to be present at provisioning time. Remove all accounts before running the installer.

  1. Settings → Accounts (or Passwords & accounts)
  2. Tap each account → Remove account
  3. Repeat until none remain

You can add accounts back after Dumbify is installed and set as Device Owner.

Setup wizard

On first launch, Dumbify guides you through 8 screens:

  1. Role — are you setting this up for yourself or someone else?
  2. Mode — Allowlist (block everything except chosen apps) or Denylist (allow everything except chosen apps)
  3. PINs — Removal PIN (mandatory, ideally held by a trusted person) and optional Bypass PIN
  4. Message — custom text shown on the block screen
  5. App picker — select which apps to allow/block; essentials (Phone, Camera, Settings) are pre-selected and locked
  6. Bypass defaults — choose friction: Delay (wait N seconds), PIN, or both
  7. Launcher — optionally replace home screen with Dumbify's minimal launcher
  8. Done — settings applied, Device Owner restrictions activated

Closing mid-wizard saves progress. The wizard resumes where you left off on next open.

Unblocking an app temporarily

When a blocked app is opened, the block screen appears. Tap a duration button (5 / 15 / 30 min) to request temporary access.

  • Delay mode — a countdown timer runs; access granted automatically when it finishes
  • PIN mode — enter the bypass PIN
  • Delay + PIN — wait out the timer, then enter the PIN

Three wrong PINs trigger a 5-minute cooldown.

Removing Dumbify

Settings → Remove Dumbify → enter Removal PIN → confirm standard Android uninstall dialog.

If you don't have the Removal PIN, the only option is a factory reset via the bootloader (power off → hold Volume Down + Power, then "Wipe data"). See FAQ: recovery.

Permissions used

Permission Why
PACKAGE_USAGE_STATS Detect which app is in foreground (to know when to block)
FOREGROUND_SERVICE AppMonitorService runs persistently to catch app launches
BIND_NOTIFICATION_LISTENER_SERVICE Suppress notifications from blocked apps
RECEIVE_BOOT_COMPLETED Restart monitor service on reboot
Device Owner (via ADB) setPackagesSuspended, setUninstallBlocked, user restrictions
